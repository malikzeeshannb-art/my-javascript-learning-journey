# JavaScript Revision README

## Day 20 — Full Revision Notes

This document records how I study JavaScript logic, how I solve problems, and where I need more repetition. It is written like a developer’s learning log, not like classroom notes.

## My Learning Style

I do not learn best by reading long theory first. I learn better when I am tested immediately.

My pattern is:

1. See a short concept.
2. Answer a direct question.
3. Get correction only where needed.
4. Move to the next step quickly.
5. Repeat the topic again later in mixed revision.

This style helps me stay active while studying instead of passively memorizing syntax.

## How I Solve Problems

When I face a JavaScript condition or logic question, I usually solve it in this order:

1. Check the values carefully.
2. Identify the operator or statement being used.
3. Ask whether the condition becomes true or false.
4. Apply operator precedence if more than one operator is present.
5. Decide what code block runs.
6. Match the result with the expected output.

This step-by-step method helps me avoid random guessing.

## What I Understand Well

These parts are becoming stronger for me:

* Basic comparison operators like `>`, `<`, `>=`, `<=`
* Strict equality with `===` and `!==`
* `if`, `if...else`, and `else if`
* Truthy and falsy values in conditions
* Simple ternary operator use
* Reading code from top to bottom and checking the final output

## Topics That Needed More Attention

These areas were more difficult at first and needed repeated practice:

* Difference between `===` and loose comparison ideas
* Logical operator order when `&&` and `||` appear together
* Operator precedence and grouping with brackets
* `switch` fall-through when `break` is missing
* Correct use of the ternary operator syntax
* Understanding how truthy/falsy values behave in real conditions

## My Common Mistakes

These are the mistakes I should watch for:

* Mixing up strict equality and loose equality logic
* Forgetting that `&&` runs before `||`
* Assuming a string like `"0"` is falsy when it is truthy
* Forgetting that `switch` continues without `break`
* Writing separate `if` statements when one `if...else` is better
* Using ternary syntax incorrectly by reversing assignment and condition
