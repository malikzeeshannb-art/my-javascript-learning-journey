This keeps the project scalable, professional, and readable.
Today I properly understood:
why we separate JavaScript files

why having an /apps folder is standard practice
3. Why <script> is Placed Before </body>
This was the part I was confused about, and today it became crystal clear.
I learned that JavaScript should run after the page loads because:
users must first see content (logo, hero section, text)
they take action only after reading
JavaScript should execute when interaction begins, not when the page loads
running JS too early slows the load speed
browser must read HTML → CSS → then JS
Now I understand the professional reason behind this placement.
